# 一、为什么选择 Hexo？

Hexo 是一个基于 Node.js 的静态博客框架，核心特点包括：

  • 使用 Markdown 写作，专注内容本身
  • 构建速度快，生成纯静态文件
  • 天然适配 GitHub Pages / Gitee Pages
  • 主题生态成熟，适合技术博客长期维护

如果你的目标是：

  • 搭建个人技术博客
  • 记录学习笔记 / 科研笔记 / 项目总结
  • 不想维护服务器

那么 Hexo 是一个非常稳妥的选择。

# 九、常见问题与建议

# 9.1 文章不显示

  • 是否忘记写 date
  • 是否文件名在 _posts/ 下
  • 是否执行了 hexo generate

# 9.2 中文路径与空格问题

  • 文件名建议使用英文或拼音
  • 图片路径统一使用相对路径

# 9.3 长期维护建议

  • 博客目录使用 Git 管理
  • source 和 _config.yml 必须纳入版本控制
  • node_modules 可不提交

# 十、总结

Hexo 的核心优势在于：

  • 写作体验接近 “纯 Markdown”
  • 构建与部署流程清晰
  • 非常适合技术与学术背景的长期记录

一旦你形成固定写作模板，维护成本会非常低。

如果你后续主要写 AI / 数学 / 代码 / 科研笔记，
建议尽早统一 Markdown 规范与公式、代码风格。
